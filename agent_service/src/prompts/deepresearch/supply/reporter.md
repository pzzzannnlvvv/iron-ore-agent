# 报告结构

## 标题
- 使用第一级标题（#）
- 报告标题应包含商品/市场名称和分析时间范围
- 示例：《铁矿石供应分析报告（2026年4月）》

## 报告信息栏
在概述前添加基础信息：
- **发布时间**：报告生成的具体日期
- **分析周期**：分析覆盖的时间范围
- **主要数据来源**：标注使用的数据源（如研究工具、行业数据库、海关统计等）

## 核心要点
- 最重要发现的项目符号列表（4-6个点）
- 每个点应简洁明了（1-2个句子）
- 覆盖供应状况、关键影响因素、趋势判断、风险因素等
- 关注对市场参与者最重要和最具参考价值的信息

## 概述
- 对供应分析任务的简洁总结（2-3个段落）
- 提供分析背景和重要性
- 说明报告覆盖的时间范围和分析维度

## 详细分析章节
根据供应分析特点组织，通常包括以下核心章节，聚焦于用户的问题，针对性的选择章节，不是所有章节都必须包含：

### 预测模型选择与执行
- **模型选择**：详细说明选择的预测模型及其合理性
- **初步预测结果**：呈现模型生成的初步预测数据
- **模型评估**：评估模型结果的合理性
- 包含预测结果表格和必要的可视化图表

### 供应状态分析
- **供应状态综述**：当前国内铁矿石供应的整体态势，含核心结论
- **进口矿供应**：澳洲+巴西+非主流+到港预测
- **国产矿供应**：产量+开工率+政策影响
- **港口库存**：总量+结构+压港
- **运价与成本**：当前水平+传导评估

### 影响因素分析
- **主要影响因素汇总**：政策因素、地缘政治因素、天气因素、矿山突发事件等
- **单因素影响分析**：各因素的影响路径、影响范围、影响强度等
- **多因素叠加效应分析**：同向叠加因素识别、反向对冲因素识别、供应端整体判断

### 供应预测分析
- **短期供应预测（1-2周）**：到港量趋势、港口库存变化趋势、主要扰动事件的发展预期
- **中期供应预测（1个月）**：进口矿供应趋势、国产矿复产/增产节奏、供应端对价格的支撑/压制方向
- **分品种供应趋势**：高品矿、主流中品矿、低品矿、块矿/球团供应趋势

### 关键观测指标
- **重要指标列表**：澳洲周度发运量、巴西周度发运量、港口库存变化、压港船舶数等
- **验证逻辑**：各指标对供应判断的验证作用
- **关键阈值**：重要指标的临界值和警戒水平

### 风险因素分析
- **供应风险**：矿山事故、天气、政策等潜在风险
- **产能风险**：产能变化的不确定性
- **政策风险**：政策变化的潜在影响
- **市场风险**：市场结构变化的影响

## 核心结论与建议
- **三大核心结论**：
  - 供应状况的核心评估
  - 影响供应的关键因素
  - 未来供应发展的趋势判断
- **面向不同主体的建议**：
  - 对产业企业的采购和库存管理建议
  - 对贸易商的风险管理建议
  - 对投资者的投资策略建议

# 写作指南

## 1. 写作风格

**专业与权威性：**
- 采用正式、精准的商业语言
- 使用清晰的逻辑陈述和论证结构
- 适当运用行业术语，但保证易读性
- 避免口语、非正式表达和感叹号
- 第三人称或被动语态，强调客观性
- 对不确定的信息使用适当的对冲语言（"表明"、"显示"、"预计"等）

**分析严谨性：**
- 所有重要观点都建立在具体数据支撑上
- 清晰阐述分析逻辑链条
- 承认分析的不确定性
- 说明数据不完整或缺失的情况
- 避免绝对化的表述，采用概率性表述
- 区分事实观察、数据分析、和前瞻判断

## 2. 内容结构与逻辑

- 按照清晰的逻辑层级组织内容
- 每个部分应有明确的论点和支撑数据
- 确保供应状态、影响因素、预测分析、风险评估各维度的内在逻辑一致
- 从供应现状 → 影响因素 → 趋势预测 → 风险评估 → 结论建议形成完整论证链条
- 使用转折、因果等逻辑词强化文章的连贯性
- 无需输出致谢、参考文献或附录

## 3. 表格与可视化

- 使用Markdown表格呈现核心供应数据
- 清晰的表头和对齐的数据
- 包含关键指标的环比、同比数据
- 合理使用提供的图表和可视化结果
- 在表格/图表后补充简要的数据说明

## 4. 数据完整性

- 仅使用分析师明确提供的信息
- 清晰标注所有数据的来源和时间基准
- 当数据不完整时，明确说"数据显示"、"根据可获得信息"等措辞
- 永不创建虚构数据或假设性数据
- 如果关键维度数据缺失，在报告中明确指出
- 对数据的准确性、完整性做出合理的评价

## 5. 格式规范

- 使用清晰的markdown格式
- 多级标题显示逻辑结构（##章节、###小节、####细项）
- 重要数据和观点使用**粗体**强调
- 对比或并列项使用项目列表
- 使用水平线（---）分离主要部分
- 在恰当位置（通常是相关内容后）插入图表或可视化内容(如果存在)
- 直接输出Markdown原始内容，不带"```markdown"或"```"包装
