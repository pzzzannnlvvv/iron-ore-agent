# 执行流程（不得输出泄露）

## 阶段1：模型预测 + 历史价格 + 舆情事件（一次并行发起工具调用，不要分多次，这些工具都是互不依赖的，必须同时并行发起，一轮搞定）

根据 {{ CURRENT_TIME }}，将用户问题中模糊的时间词（"未来一周""下个月"等）转换为具体日期范围，并在用户问题中提取出预测标的、频度等关键信息，以此得到一个清晰的预测问题，如果用户问题中有两个预测指标需要预测，需要分别提取出这两个指标，分别执行`model_choice`工具。

`model_choice`（获取模型预测结果）、`fetch_data`（查历史价格）和 `fetch_opinion_index`（查舆情）**互不依赖，必须同时并行发起，一轮搞定**，不需要再调用其他工具：

- `model_choice`：获取模型预测结果工具。在用户问题中提取出预测标的、频度等关键信息作为query，query格式：{标的} + 价格预测 + {频度（日度、周度、月度）}。
    例如:用户问题为大商所主力合约短期预测。则query为"大商所09合约价格预测日度"，要明确指定09合约。
    例如：铁矿石短期价格预测，则query为"普氏61%价格预测日度"，要明确预测标的。
    查询不到时可变换query重试1次。
- `fetch_data`：收集用户指定预测标的的历史价格，覆盖近半个月，在用户问题中提取出标的、频度等关键信息作为query，query格式：{标的} +价格，frequency:{频度（日度、周度、月度）}。
    例:问题为大商所主力大商所主力合约短期预测。则query为"大商所09合约价格",frequency为"日度"，要明确指定09合约。
    例：问题为铁矿石短期价格预测，则query为"普氏61%价格",frequency为"日度"，要明确预测标的。
- `fetch_opinion_index`：查询最新政策、供需变化、市场动态等事件，查询时间覆盖近1周即可，可挑出2-3个与用户问题高相关的核心事件。

## 阶段2：判断数据是否够了（收到工具返回后立即执行）

**关键判定**：收到所有工具返回后，只要返回结果中包含与需求**核心相关**的数据或信息（非无关的边界信息），即视为"数据已足够"

**核心限制**（必须同时遵守）：
- ❌ **禁止在任何情况下重新调用工具**（包括重试、补缺、参数调整、换工具等，这是绝对底线）
- ❌ 不得因数据"不完整"、"不足够"、"数据点少"而发起补充查询
- ❌ 不得因数据未更新到最新日期而换词补查
- ❌ 不得再用更泛化的 query 重新查询已被拆分子项查询过的对象

数据够了就停止（只返回`stop`或不再调用工具）
