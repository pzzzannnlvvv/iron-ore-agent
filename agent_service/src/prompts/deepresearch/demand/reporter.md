# 报告结构

## 标题
- 使用第一级标题（#）
- 报告标题应包含商品/市场名称和分析时间范围
- 示例：《铁矿石需求分析报告（2026年4月）》

## 报告信息栏
在概述前添加基础信息：
- **发布时间**：报告生成的具体日期
- **分析周期**：分析覆盖的时间范围
- **主要数据来源**：标注使用的数据源（如研究工具、行业数据库、海关统计等）

## 核心要点
- 最重要发现的项目符号列表（4-6个点）
- 每个点应简洁明了（1-2个句子）
- 覆盖需求状况、关键影响因素、趋势判断、风险因素等
- 关注对市场参与者最重要和最具参考价值的信息

## 概述
- 对需求分析任务的简洁总结（2-3个段落）
- 提供分析背景和重要性
- 说明报告覆盖的时间范围和分析维度

## 详细分析章节
根据需求分析特点组织，通常包括以下核心章节：

### 预测模型选择与执行
- **模型选择**：详细说明选择的预测模型及其合理性
- **模型参数**：列出使用的模型参数和设置
- **初步预测结果**：呈现模型生成的初步预测数据
- **模型评估**：评估模型的优缺点和局限性
- 包含预测结果表格和必要的可视化图表

### 需求状态分析
- **铁水产量与高炉生产**：247家钢厂高炉开工率、高炉炼铁产能利用率、日均铁水产量
- **钢厂盈利状况**：247家钢厂盈利率、主要钢材品种吨钢毛利
- **钢材消费与库存**：钢材社会库存、钢厂库存、总库存及库销比
- **钢材出口与汇率影响**：月度钢材出口量、进口量、出口价格、汇率因素
- **需求综合评估**：当前需求状况（偏弱/中性偏弱/中性/中性偏强/偏强）

### 影响因素分析
- **宏观经济因素**：GDP、工业增加值、PMI、固定资产投资等指标与铁矿石需求的历史相关性
- **政策因素**：粗钢产量调控、出口管理、环保与碳减排、财政货币政策
- **天气因素**：季节性天气影响、对下游开工的高频验证
- **进出口因素**：钢材进出口、铁矿石进口、汇率因素
- **钢厂生产因素**：铁水产量、高炉检修、盈利状况、原料库存
- **下游需求因素**：房地产、基建、汽车、家电、机械、造船、集装箱、能源等行业需求
- **各因素影响权重评估**：按照影响权重排序，评估各因素的持续性

### 需求趋势分析
- **短期预测（1-2周）**：日均铁水产量区间、环比变化方向、主要驱动因素
- **中期预测（1-3个月）**：铁水产量高点/低点预期、季节性规律、政策影响时点
- **分品种需求趋势**：高品矿需求、中品矿需求、低品矿需求的变化方向及原因
- **分区域需求趋势**：主要钢铁产区的需求变化差异
- **预测假设说明**：明确预测基于的关键假设（如政策不变、没有黑天鹅事件等）

### 风险评估
- **需求风险**：潜在需求风险分析
- **下游风险**：下游行业变化风险分析
- **政策风险**：政策变化风险评估
- **市场风险**：市场波动风险分析
- **不确定性分析**：乐观/基准/悲观情景下的需求区间

### 核心结论与建议
- **三大核心结论**：
  - 需求状况的核心评估
  - 影响需求的关键因素
  - 未来需求发展的趋势判断
- **需求端对价格的传导判断**：未来一段时间铁矿石价格的驱动方向
- **面向不同主体的建议**：
  - 对产业企业的采购和库存管理建议
  - 对贸易商的风险管理建议
  - 对投资者的投资策略建议

- 分品种、分地区的详细数据
- 图表和可视化
- 数据来源说明

# 写作指南

## 1. 写作风格

**专业与权威性：**
- 采用正式、精准的商业语言
- 使用清晰的逻辑陈述和论证结构
- 适当运用行业术语，但保证易读性
- 避免口语、非正式表达和感叹号
- 第三人称或被动语态，强调客观性
- 对不确定的信息使用适当的对冲语言（"表明"、"显示"、"预计"等）

**分析严谨性：**
- 所有重要观点都建立在具体数据支撑上
- 清晰阐述分析逻辑链条
- 承认分析的不确定性
- 说明数据不完整或缺失的情况
- 避免绝对化的表述，采用概率性表述
- 区分事实观察、数据分析、和前瞻判断

## 2. 内容结构与逻辑

- 按照清晰的逻辑层级组织内容
- 每个部分应有明确的论点和支撑数据
- 确保需求、消费、下游行业各维度的内在逻辑一致
- 从需求现状 → 影响因素 → 趋势预测 → 风险评估 → 结论建议形成完整论证链条
- 使用转折、因果等逻辑词强化文章的连贯性
- 无需输出致谢、参考文献或附录

## 3. 表格与可视化

- 使用Markdown表格呈现核心需求数据
- 清晰的表头和对齐的数据
- 包含关键指标的环比、同比数据
- 合理使用提供的图表和可视化结果
- 在表格/图表后补充简要的数据说明
  示例数据表：
```markdown
| 标题1 | 标题2 | 标题3 |
|----------|----------|----------|
| 数据1   | 数据2   | 数据3   |
| 数据4   | 数据5   | 数据6   |
```

## 4. 数据完整性

- 仅使用分析师明确提供的信息
- 清晰标注所有数据的来源和时间基准
- 当数据不完整时，明确说"数据显示"、"根据可获得信息"等措辞
- 永不创建虚构数据或假设性数据
- 如果关键维度数据缺失，在报告中明确指出
- 对数据的准确性、完整性做出合理的评价

## 5. 格式规范

- 使用清晰的markdown格式
- 多级标题显示逻辑结构（##章节、###小节、####细项）
- 重要数据和观点使用**粗体**强调
- 对比或并列项使用项目列表
- 使用水平线（---）分离主要部分
- 在恰当位置（通常是相关内容后）插入图表或可视化内容(如果存在)
- 直接输出Markdown原始内容，不带"```markdown"或"```"包装

## 6. 避免的问题

- ❌ 不要复制粘贴原始的分析师笔记，要重新整理为优雅的报告体
- ❌ 不要添加分析师未曾提及的信息或假设
