## 三、概念说明

### 试验次数（Trials）

Optuna 每次「猜」一组超参数 → 用这组参数训练模型 → 在验证集上打分 → 记录下来，这整个过程叫 **1 次试验**。

- 50 次 = 快速验证流程是否跑通
- 200 次 = 正式搜索，找到更优的参数

### train / val / test 的用途

```
Optuna 优化阶段（optuna_lgbm_optimize.py）
  train.csv ──→ 训练模型
  val.csv   ──→ 验证效果（Optuna 根据这个分数调整搜索方向）
  test.csv  ──→ ❌ 完全不碰！（最后的考试卷，不能偷看）

复现推理阶段（lgbm_infer_reproduce.py）
  train.csv + val.csv ──→ 用最优参数重新训练
  test.csv            ──→ 最终考试，得出真正的 MDA/MAPE
```

## 五、执行步骤

### Step 1: 创建 optuna_lgbm_optimize.py

1. 加载 train.csv 和 val.csv，分离特征 X 和目标 y
2. 设定全局随机种子（42），确保可复现
3. 定义 MDA 计算函数：
   - 方向正确 = sign(预测变化) == sign(实际变化)
4. 定义 objective(trial) 函数：
   - trial 建议超参 → 训练 LightGBM → 在 val 上计算 MDA → 返回 1-MDA（转为最小化）
5. 创建 Optuna study（TPESampler + MedianPruner）
6. 运行 optimize（50 次快速验证 → 确认无误 → 200 次正式跑）
7. 保存：CSV + best_params.json + optimization_summary.json

**9 个搜索的超参**：

| 参数 | 范围 | 分布 |
|------|------|------|
| num_leaves | 15 ~ 255 | 整数均匀 |
| max_depth | 3 ~ 15 | 整数均匀 |
| learning_rate | 0.01 ~ 0.3 | log-uniform |
| n_estimators | 100 ~ 2000 | 整数均匀 |
| min_child_samples | 5 ~ 100 | 整数均匀 |
| subsample | 0.5 ~ 1.0 | 浮点均匀 |
| colsample_bytree | 0.5 ~ 1.0 | 浮点均匀 |
| reg_alpha | 1e-8 ~ 10.0 | log-uniform |
| reg_lambda | 1e-8 ~ 10.0 | log-uniform |

### Step 2: 创建 lgbm_infer_reproduce.py

1. 加载 train.csv + val.csv 合并为训练集，test.csv 为测试集
2. 读取 best_params.json 的最优超参
3. 用相同随机种子（42）重新训练
4. 对 test.csv 预测，计算 MDA 和 MAPE
5. 保存 test_predictions.csv

### Step 3: 验证

1. 跑 optuna_lgbm_optimize.py（50 trials），确认无报错
2. 跑 lgbm_infer_reproduce.py，确认复现结果一致
