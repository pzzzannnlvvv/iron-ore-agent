# knowledge_service 复刻笔记(对照原版 knowledge_be)

> 原版 `xmschain_knowledge_be` 是 Java 服务(ES + MyBatis-Plus + RocketMQ + JanusGraph…)。
> 复刻版用 Python 只做 RAG 检索核心,跳过基础设施,聚焦学习。

## 复刻了什么(逐项对照)

| 原版 knowledge_be(Java) | 复刻版 knowledge_service(Python) | 对照说明 |
|---|---|---|
| `KnowledgeSearchServiceImpl`(845 行,主链路) | `retrieval/search.py` | 三模式召回 + 降级 + rerank + 三段耗时,逻辑对齐 |
| ES 向量检索(KNN) | `pipeline/store.py` Chroma KNN(cosine) | 简化:Chroma 本地文件替 ES,零运维 |
| ES BM25 | `retrieval/bm25.py` rank_bm25 + jieba | 简化:内存 BM25,非 ES |
| `EmbeddingRerankApi`(HTTP 调外部 rerank) | `pipeline/reranker.py`(本地 bge-reranker) | 简化:本地推理,少一个服务 |
| bge-m3 embedding(外部服务) | `pipeline/embedder.py`(本地 BGEM3FlagModel) | 同款模型,本地推理 |
| `FileContentSplitter`(段落/表格/长度多策略) | `pipeline/splitter.py`(递归字符切片) | 简化:单一递归策略,无表格识别 |
| `application-dev.yml` splitter/embedding/rerank 配置 | `src/config.py` + `.env` | 参数化,默认值参考原版 |
| ES mappings `kbIronOreCore.json` 字段设计 | Chroma metadata(fileName/source/sourceType/chunkIndex) | 对齐核心检索字段 |
| 三段耗时日志(embedding/search/reRank) | `timings` dict + `logger` | 对齐 |
| 降级:embedding 失败 -> 纯 BM25 | `search.py` 同 | 对齐 |
| 检索结果 `KnowledgeSearchVO` | `_to_vo` 输出同字段 | 对齐(id/content/fileName/source/score…) |

## 简化了什么 + 为什么

1. **ES -> Chroma**:学习重点是 RAG 链路,不是 ES 运维。Chroma 本地文件,零部署。
2. **JanusGraph 图谱 / 舆情 / 专题数据**:跳过(`fetch_graph`/`fetch_news` 保持占位),先做"用户上传"知识库这一条主路。
3. **RocketMQ / S3(EOS)/ Quartz / 多数据源 / MyBatis-Plus**:基础设施,非 RAG 学习重点,全跳。
4. **表格识别切片**:原版 `FileContentSplitter` 会识别表格单独切;复刻版用通用递归字符切片,够用,后续要更准可加表格识别。
5. **Rerank 由 HTTP 服务 -> 本地推理**:原版调外部 rerank 服务;复刻版本地加载 bge-reranker,少一个服务进程。

## 与原版语义对齐(关键)

embedding 和 rerank 都用 **bge 同款模型**(bge-m3 / bge-reranker-v2-m3),所以检索语义和原版一致。这是"复刻"能成立的关键——换 doubao-embedding 等其他模型会破坏语义对齐,失去复刻意义。

## 验证结果

- **hit@5 = 88%**(8 个 query,`scripts/eval.py`)
- **reRank=1 比 reRank=0 更准**:同一 query,reRank=0 顶回来的是泛泛流程总结(RRF 分);reRank=1 顶回来的是报告核心结论片段(reranker sigmoid 分)。
- **三段耗时日志**正常:`embeddingTimeCost` / `searchTime` / `reRankTimeCost` / `totalHits`。
- **降级**:`embed()` 抛异常时自动退化为纯 BM25(照搬原版)。

## 性能(CPU,本机无 GPU)

- 首次加载:bge-m3 ~10s、bge-reranker ~3s(各 2.27G 权重入内存)
- 单次检索:embedding 0.1s(热)+ 召回 0.01s + rerank ~10s(20 候选逐对打分)
- rerank 是瓶颈。学习够用;要提速可减小候选池或上 GPU。
