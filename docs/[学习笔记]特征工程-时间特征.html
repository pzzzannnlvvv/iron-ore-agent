二、分模块详解

2.1 基础时间特征（7个）

year, month, day, quarter, dayofweek, dayofyear, weekofyear
提取日期最基本的组成部分，作为模型的基础输入。
概念解释

基础时间特征：直接从日期对象中提取各个时间分量，把日期拆解成数字，让模型可以理解和使用。

  • year（年份）：比如2023、2024
  • month（月份）：1-12
  • day（日期）：1-31
  • quarter（季度）：1-4
  • dayofweek（周几）：0=周一，6=周日
  • dayofyear（一年中的第几天）：1-365/366
  • weekofyear（一年中的第几周）：1-52/53
代码实现
features['year'] = date_col.dt.year.values
features['month'] = date_col.dt.month.values
features['day'] = date_col.dt.day.values
features['quarter'] = date_col.dt.quarter.values
features['dayofweek'] = date_col.dt.dayofweek.values
features['dayofyear'] = date_col.dt.dayofyear.values
features['weekofyear'] = date_col.dt.isocalendar().week.values
逐行解释
date_col.dt.year.values
小白理解：从日期列里取出年份，转换成 numpy 数组
date_col.dt.month.values
小白理解：从日期列里取出月份（1-12）
date_col.dt.day.values
小白理解：从日期列里取出日期（1-31）
date_col.dt.quarter.values
小白理解：从日期列里取出季度（1-4）
date_col.dt.dayofweek.values
小白理解：从日期列里取出周几（0=周一，6=周日）
date_col.dt.dayofyear.values
小白理解：从日期列里取出是一年中的第几天（1-365/366）
date_col.dt.isocalendar().week.values
小白理解：从日期列里取出是一年中的第几周

2.2 月度哑变量（11个）

month_1 ~ month_11
把月份变成 0/1 特征，避免线性共线性只保留 1-11 月。
概念解释

哑变量（Dummy Variables）：也叫虚拟变量，把一个分类变量转换成多个 0/1 变量。

  • 比如「月份」有 12 个类别，不能直接输入 1-12，因为模型会误以为 12 比 1「大」
  • 正确做法是每个月单独一列：是1月=1，不是=0；是2月=1，不是=0，以此类推

共线性问题：12个月的哑变量加起来恒等于1，存在完美共线性，会导致模型无法求解，所以要去掉一个（通常去掉最后一个）。

  • 只保留 1-11月
  • 12月用「所有列都是0」来表示
代码实现
for m in range(1, 12):
    name = f'month_{m}'
    features[name] = (date_col.dt.month == m).astype(np.float32).values
逐行解释
for m in range(1, 12):
小白理解：循环 m 从 1 到 11（注意 range 是左闭右开，所以不包含 12）
name = f'month_{m}'
小白理解：构造特征列名，比如 m=1 时就是 'month_1'
date_col.dt.month == m
小白理解：判断每一行的月份是不是等于 m，得到 True/False 数组
.astype(np.float32)
小白理解：把 True/False 变成 1.0/0.0
.values
小白理解：转成 numpy 数组存到 features 字典里

2.3 季度哑变量（3个）

quarter_1 ~ quarter_3
同理，季度哑变量也只取 1-3 季度。
概念解释

季度哑变量：和月度哑变量同理，把季度也转换成 0/1 特征。

  • quarter_1：第一季度（3月）=1，否则=0
  • quarter_2：第二季度（4-6月）=1，否则=0
  • quarter_3：第三季度（7-9月）=1，否则=0
  • quarter_4：第四季度（10-12月）用「所有列都是0」表示

季度通常和季节效应、季度业绩等有关。

代码实现
for q in range(1, 4):
    name = f'quarter_{q}'
    features[name] = (date_col.dt.quarter == q).astype(np.float32).values
逐行解释
for q in range(1, 4):
小白理解：循环 q 从 1 到 3（不包含 4）
name = f'quarter_{q}'
小白理解：构造特征列名，比如 q=1 时就是 'quarter_1'
date_col.dt.quarter == q
小白理解：判断每一行的季度是不是等于 q
.astype(np.float32).values
小白理解：转成 1.0/0.0 的 numpy 数组

2.4 正弦/余弦周期编码（4个）

month_sin, month_cos, quarter_sin, quarter_cos
把周期性特征（月份、季度）编码成连续的圆形特征，让模型感知“12月离1月很近”。
概念解释

周期编码（Cyclic Encoding）：用正弦和余弦函数把周期特征映射到一个圆上。

  • 如果只用 1-12 的数字，模型会觉得 12月离 1月很远，但实际上它们在时间上是相邻的
  • 用圆表示后，12月和1月在圆上是邻居
❌ 线性表示的问题
✅ 圆形表示的好处

为什么要用两个值（sin+cos）？

  • 只用一个值（比如sin）的话，圆上会有两个点对应同一个值，无法区分
  • sin + cos 可以唯一确定圆上的一个点
代码实现
features['month_sin'] = np.sin(2 * np.pi * date_col.dt.month.values / 12)
features['month_cos'] = np.cos(2 * np.pi * date_col.dt.month.values / 12)
features['quarter_sin'] = np.sin(2 * np.pi * date_col.dt.quarter.values / 4)
features['quarter_cos'] = np.cos(2 * np.pi * date_col.dt.quarter.values / 4)
逐行解释
2 * np.pi
小白理解：就是一个完整的圆（360度）
date_col.dt.month.values / 12
小白理解：把月份归一化到 0-1 之间（1月≈0.08，12月≈1.0）
np.sin(...)
小白理解：计算正弦值，把月份映射到圆的 Y 坐标
np.cos(...)
小白理解：计算余弦值，把月份映射到圆的 X 坐标

2.5 春节标记（3个）

spring_before_15d, spring_during, spring_after_15d
春节是国内最重要的节日，会影响钢厂生产、物流、库存行为，因此单独标记。
概念解释

节日效应：春节期间，工厂可能会放假、物流可能会停滞、下游需求可能会提前备货或延后，因此需要单独标记。

  • spring_before_15d：春节前15天到前一天，通常是备货期
  • spring_during：春节当天到后6天，通常是假期，生产和物流停滞
  • spring_after_15d：春节后7天到22天，通常是恢复生产期

0/1标记：是这段时间=1，不是=0。

代码实现
# 先创建 3 个全 0 数组
sf_before = np.zeros(n, dtype=np.float32)
sf_during = np.zeros(n, dtype=np.float32)
sf_after = np.zeros(n, dtype=np.float32)

# 遍历每一年的春节日期
for year, sf_date_str in spring_festival.items():
    sf_date = pd.Timestamp(sf_date_str)
    # 前15天到前一天
    sf_before[(date_col >= sf_date - pd.Timedelta(days=15)) & (date_col < sf_date)] = 1.0
    # 春节当天到后6天
    sf_during[(date_col >= sf_date) & (date_col < sf_date + pd.Timedelta(days=7))] = 1.0
    # 后7天到后22天
    sf_after[(date_col >= sf_date + pd.Timedelta(days=7)) & (date_col <= sf_date + pd.Timedelta(days=22))] = 1.0
春节节前窗口的逐行解释
sf_before = np.zeros(n, dtype=np.float32)
小白理解：先创建一个全是 0 的数组，默认都不在节前
sf_date - pd.Timedelta(days=15)
小白理解：春节初一往前数 15 天（节前窗口的起点）
date_col >= ...
小白理解：日期要在这一天之后或当天
date_col < sf_date
小白理解：日期还要早于初一（初一当天不算节前）
&
小白理解：两个条件同时满足
sf_before[...] = 1.0
小白理解：满足的行，在 sf_before 里标成 1
时间窗口总结
  • spring_before_15d：春节前 15 天到前一天
  • spring_during：春节当天到后 6 天
  • spring_after_15d：春节后 7 天到 22 天

2.6 采暖季标记（1个）

heating_season
北方采暖季从 11月15日 到次年 3月15日，期间钢铁需求与产能会受到影响。
概念解释

采暖季效应：中国北方冬季会集中供暖（通常从11月15日到3月15日）。

  • 供暖需求增加会推升动力煤、天然气等能源价格
  • 环保限产可能会影响钢铁产能
  • 建材需求在冬季可能会有淡季效应

因此需要单独标记采暖季，让模型学习这个时期的特殊规律。

代码实现
heating = np.zeros(n, dtype=np.float32)
for i, dt in enumerate(date_col):
    m = dt.month
    d = dt.day
    # 12月、1月、2月：整月采暖
    if m > 11 or m < 3:
        heating[i] = 1.0
    # 11月15日及之后
    elif m == 11 and d >= 15:
        heating[i] = 1.0
    # 3月15日及之前
    elif m == 3 and d <= 15:
        heating[i] = 1.0
采暖季代码的逐行解释
heating = np.zeros(n, dtype=np.float32)
小白理解：先创建一个全是 0 的数组，默认都不在采暖季
for i, dt in enumerate(date_col):
小白理解：一行一行遍历日期，i是行号，dt是这一行的日期
m = dt.month
小白理解：取出当前这一行的月份
d = dt.day
小白理解：取出当前这一行的日期
if m > 11 or m < 3:
小白理解：月份大于 11（12月）或者小于 3（1月、2月）
heating[i] = 1.0
小白理解：满足的话，这一行标成 1（属于采暖季）
elif m == 11 and d >= 15:
小白理解：如果是 11 月且日期 >= 15 日
elif m == 3 and d <= 15:
小白理解：如果是 3 月且日期 <= 15 日
采暖季范围总结
  • 11月15日 ~ 11月30日
  • 12月1日 ~ 2月28/29日（整月）
  • 3月1日 ~ 3月15日

2.7 月初/月末标记（2个）

month_start, month_end
月初（1-5日）和月末（25-31日）常伴随结算、补库行为，可能影响库存变化。
概念解释

月初月末效应：每月的月初和月末常伴随一些特殊的商业行为。

  • 月末（25-31日）：可能有月度结算、考核、月底冲量等行为
  • 月初（1-5日）：可能有新一月的计划、补货、资金回笼等

这些时点的库存、价格、需求可能会和平时不一样，所以单独标记。

代码实现
features['month_start'] = (date_col.dt.day <= 5).astype(np.float32).values
features['month_end'] = (date_col.dt.day >= 25).astype(np.float32).values
逐行解释
date_col.dt.day <= 5
小白理解：判断每一行的日期是不是 <= 5（1-5日）
.astype(np.float32)
小白理解：把 True/False 变成 1.0/0.0
.values
小白理解：转成 numpy 数组
features['month_end']
小白理解：月末同理，判断日期是不是 >= 25（25-31日）
