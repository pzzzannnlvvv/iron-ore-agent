🗝️ 关键概念说明

MDA (Mean Directional Accuracy)

方向准确率：预测涨跌方向的正确率

def mda(y_true, y_pred): true_dir = np.sign(np.diff(y_true)) pred_dir = np.sign(np.diff(y_pred)) return np.mean(true_dir == pred_dir)

✓ 核心业务指标：方向对了，交易决策才有意义

MAPE (Mean Absolute Percentage Error)

平均绝对百分比误差：相对误差大小

def mape(y_true, y_pred): return np.mean(np.abs((y_true - y_pred) / y_true))

⚠️ 注意：sklearn输出是小数，显示时需×100

🏷️ 五大特征类别

3184个特征从构建方式来看，可归为5大类：

①
时间日历特征（Step 2）
从日期中提取，不依赖业务数据。year、month、quarter、dayofweek、month_sin/cos（周期性编码）、春节前15天标记（备货期）、采暖季标记（需求旺季）
作用：让模型捕捉季节性规律（冬季补库、年末冲量等）
②
滞后与变化率特征（Step 3 + 4 + 8）
把时间轴上的"过去值"变成特征，是最核心的一类。包含标的自身滞后（lag_1~365, diff, pct）、因子滞后（核心因子 lag_1/7/30 + diff + pct）、同比环比（YoY/MoM/WoW）
作用：让模型看到"上周/上月/去年同期发生了什么"以及"变化了多少"
③
趋势与波动特征（Step 5）
对原始值做滚动窗口统计。移动平均（MA3~90）、移动标准差（STD7~60）、移动分位数（P10/P50/P90）、动量指标（MOM5/10/20 = 当前值/过去均值）
作用：让模型理解"在上升还是下降通道""波动是否在放大"
④
技术指标特征（Step 7）
借鉴金融市场的技术分析方法，主要用于价格类数据。均线类（Price/MA5, MA5/MA20 金叉死叉信号）、震荡类（RSI7/14 超买超卖）、波动类（ATR 真实波幅、布林带）
作用：捕捉价格的趋势反转信号和波动状态
⑤
业务逻辑衍生特征（Step 6）
需要领域知识手动构建，不可自动推导。供应侧（四大矿山发运占比、到港量变化）、需求侧（废钢/铁水比价、产能利用率）、价格侧（普钢-铁矿价差、高炉利润）、供需平衡（库存周转天数）
作用：注入行业专家知识，让模型理解"为什么涨/跌"而不只是"涨/跌了多少"

💡 特征构成比例

②③④ 本质上都是对原始值做不同窗口的数学变换，贡献了绝大部分特征数量（~3000+）；①⑤ 数量少但信息密度高，属于不可替代的先验知识。

自动推导类（②③④）

~3100+ 特征，通过数学变换自动生成，覆盖各种窗口和统计量

先验知识类（①⑤）

~50 特征，数量少但包含不可替代的行业洞察和时间规律

🔻 特征筛选三层漏斗（7112 → 3184）

1
保护层 Protected
以下三类全部无条件保留：① 时间特征（year, month, quarter, spring_*, heating_*）② 目标特征（target_lag_*, diff_*, pct_* — 16列）③ 库存因子衍生列（基列 ID00186052~00186060 的所有衍生）
2
组内去重 Within-Group Dedup
按基础因子 ID 分组 → 按方差降序排列 → 若与已保留列 |Pearson r| > 0.98 则丢弃 → 组内只保留方差最高 + 不高度相关的列
3
LightGBM 重要性截断（条件触发）
若前两层后仍超过目标列数，用 LGBM 快速训练（200棵树），按 feature_importance 保留 Top-N。本轮实际：前两层后已降至 3184 列，第三层未触发。

⚙️ Optuna 超参搜索空间（9 个参数）

覆盖 LGBM 的树结构、训练策略、正则化三大维度。其中 learning_rate 和 reg_* 使用 log-uniform 分布——在低值区更密集采样，因为小数变化对效果影响更大。

参数 范围 分布 作用
num_leaves15 ~ 255整数均匀叶节点数，控制树复杂度
max_depth3 ~ 15整数均匀树深度上限
learning_rate0.01 ~ 0.3log-uniform学习率
n_estimators100 ~ 2000整数均匀树的数量
min_child_samples5 ~ 100整数均匀叶节点最少样本数（正则）
subsample0.5 ~ 1.0浮点均匀数据行采样比例
colsample_bytree0.5 ~ 1.0浮点均匀特征列采样比例
reg_alpha1e-8 ~ 10.0log-uniformL1 正则化
reg_lambda1e-8 ~ 10.0log-uniformL2 正则化

🎯 关键设计决策

决策项 选择 原因
优化目标MDA（方向准确率）库存预测核心诉求是涨跌方向
试验次数200 次先 50 验证再 200 正式；实测 100 次后边际收益归零
采样器TPESampler（贝叶斯）利用历史结果指导搜索，比随机收敛更快
剪枝器无（周频版移除）时序 CV 各折样本少，剪枝不可靠
验证方式时序交叉验证（Time Series CV）不设独立验证集，train 内部严格按时间顺序交叉验证，充分利用有限数据
最终评估独立 test_weekly.csv调参全程不加载，只在复现推理阶段首次评估
数据隔离14 周 Gap防止时间信息泄露
可复现性SEED=42 全局统一训练 + 拆分 + Optuna 全锁定

🧠 方法论总结：各环节重要性分析

从项目实践中提炼的经验：模型最终表现由数据质量、特征工程、模型选择、超参调优四个环节共同决定，但它们的贡献极不均衡。

特征工程
~70%
数据质量
~20%
模型选择
~5%
超参调优
~5%

🟢 特征工程决定上限（~70%）

  • 20个业务特征注入了"矿山发运↑→到港量↑→库存↑→价格↓"这类因果链，这是模型无法自己学到的
  • 3000+个滞后/趋势/技术指标特征覆盖了不同时间尺度的信号
  • 三层漏斗筛选（7112→3184）去掉了噪声，保留了有效信息
  • 周频聚合策略（取周五）让日频信号对齐周频目标

🔵 数据质量是地基（~20%）

  • 清洗前三个问题标的数据频繁跳变到0
  • 14周 Gap 隔离带防止时间泄露
  • IQR截断 + 跳变检测保证数据可靠性
  • 前向填充对齐多频率数据（日/周/月/季/年）

🟡 模型选择是锦上添花（~5%）

LightGBM 对表格数据本身就是强 baseline，默认参数就能达到不错的水平。模型选择的差异远小于特征质量的差异。

🟡 超参调优边际递减（~5%）

Optuna 200次试验中，100次后 MDA 收益归零。从50→200次（4倍投入）仅换来 +1.89% MDA 提升。超参不是瓶颈。

📌 佐证：问题标的的启示

三个问题标的（超特粉_曹妃甸港、超特粉_江阴港、卡拉加斯粉_江阴港）的测试 MDA 只有 51~58%，远低于主标的 84.91%。它们用的是同一套模型和超参，唯一的区别是数据质量差（频繁跳变到0）。这说明当数据/特征有问题时，再好的模型也救不回来。

一句话：模型决定下限，特征决定上限。

💡 后续优化建议

  • 三个问题标的可尝试用 滚动中位数 或 前向填充 替代 Winsorize
  • 可考虑 滚动训练 (rolling training) 提升模型对新数据的适应性
  • 可尝试 集成学习：多模型融合提升稳定性
