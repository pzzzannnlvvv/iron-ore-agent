二、数据清洗三大利器

校验类型 判断标准 处理方法 业务逻辑
1. 库存负值 value < 0 设为缺失值（NaN） 库存不可能为负，明显数据错误
2. 下界异常值 value < Q1 - 3×IQR 截断到下界 保留趋势，避免极端值影响
3. 上界异常值 value > Q3 + 3×IQR 截断到上界 保留趋势，避免极端值影响
4. 异常跳变 |Δt| > 5×σ(Δ) 设为缺失值（NaN） 单日跳变过大，可能是数据错误

三、上下界异常值（IQR方法）详解

概念解释

IQR（Interquartile Range，四分位距）是一种稳健的异常值检测方法，不受极端值影响。

计算公式

数学公式
Q1 = 第25百分位数 （数据从小到大排，第25%位置的值）
Q3 = 第75百分位数 （数据从小到大排，第75%位置的值）
IQR = Q3 - Q1 （中间50%数据的范围）
下界 = Q1 - 3 × IQR
上界 = Q3 + 3 × IQR

实际例子

铁矿石库存数据示例

数据序列：100, 102, 105, 110, 95, 98, 103, 9999（异常大值）

Q1 = 98，Q3 = 105，IQR = 7

下界 = 98 - 3×7 = 77，上界 = 105 + 3×7 = 126

发现：9999 > 126 → 上界异常值

修正方式：截断（Winsorize）

不是删除，而是改成边界值！这样既保留了数据趋势，又避免极端值过度影响模型。
原值：9999 → 修正为：126

为什么用3×IQR而不是1.5×IQR？

  • 1.5×IQR：太严格，容易误杀正常波动
  • 3×IQR：更宽松，只抓真正极端的异常值

四、异常跳变检测详解

概念解释

关注的不是绝对值大小，而是相邻两天的变化量。

实际例子

异常跳变示例

第1天：100
第2天：102（正常，+2）
第3天：105（正常，+3）
第4天：9999（不正常，+9894）→ 跳变！
第5天：101（又回来了）→ 跳变！

计算方式

计算步骤
  1. 先计算每天的变化量 Δt = 第t天 - 第t-1天
  2. 计算变化量的标准差 σ(Δ)
  3. 如果 |Δt| > 5×σ(Δ) → 视为异常跳变

修正方式：设为缺失值

这种"过山车"式的数据明显是录入错误，直接设为空值，后续可以前向填充。
第4天：9999 → 修正为：NaN
第5天：101 → 修正为：NaN

六、特征工程策略：不要对700列盲目做特征！

分层处理方案

列类型 数量 特征工程内容
标的列（Y） ~27列 只做滞后项（t-1, t-2, t-4, t-8, t-12）
核心因子 ~30个 滞后、移动平均、波动率、同比环比
次因子 ~50个 只做滞后项
其他因子 ~600个 暂时不做，或直接用原始值

从简到繁的迭代策略

  1. 第一版：只用原始值（不做复杂特征工程），快速验证baseline
  2. 第二版：逐步添加，先加标的滞后，再加核心因子特征
  3. 第三版：看效果提升再继续扩展
