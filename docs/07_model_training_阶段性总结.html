2.2 特征合并 — Step 9

  1. 删除全 NaN 列
  2. 删除缺失率 > 50% 列

  4. 剩余 NaN：前向填充 → 后向填充

输出：08_features_merged.csv

2.3 特征筛选 — Step 10（核心）

10_feature_selection.py — 三层漏斗，逐层精炼：

第一层 · 保护层 Protected

以下三类全部无条件保留：
① 时间特征（year, month, quarter, spring_*, heating_*）
② 目标特征（target_lag_*, diff_*, pct_* — 16列）
③ 库存因子衍生列（基列 ID00186052~00186060 的所有衍生）

第二层 · 组内去重 Within-Group Dedup

按基础因子 ID 分组（如 ID02017483 的 lag_7、MA30、yoy_365 归为一组）：
→ 按方差降序排列
→ 若与已保留列 |Pearson r| > 0.98，则丢弃
→ 组内只保留方差最高 + 不高度相关的列

第三层 · LightGBM 重要性截断 （条件触发）

若前两层后仍超过目标列数，用 LGBM 快速训练（200 棵树），按 feature_importance 保留 Top-N。

本轮实际：前两层后已降至 3184 列（在目标 2700~3700 内），第三层未触发。

数据使用说明：整个流程只使用两份磁盘文件：train_weekly.csv 和 test_weekly.csv。不存在独立的 val.csv。
Optuna 调参期间：只加载 train，每轮 trial 在 train 内部按时间顺序切 90%/10%——前 90% 训练 LGBM，后 10% 计算 MDA 作为 Optuna 优化信号。test_weekly.csv 完全不加载。
复现推理阶段：用最优参数在 train 全量上重训，首次加载 test 做最终评估。输出「测试 MDA / MAPE」。

Step 2：超参优化

核心思路：Optuna 每次「猜」一组超参 → 从 train 内部临时切 10% 作为验证 → 训练 LGBM → 验证 MDA → 贝叶斯采样器根据分数调整下一组猜测方向。200 次迭代后输出最优参数。全过程不碰 test.csv。

数据加载与目标构造

从 train_weekly.csv 加载双表头数据，利用 target_lag_1 向后平移 1 周构造预测目标 _y（即本周实际库存值），然后移除所有 target_* 列避免数据泄露。

📄 加载与目标构造 ▼ 
train = pd.read_csv("train_weekly.csv", header=[0, 1])
train["_y"] = train["target_lag_1"].shift(-1)   # y[t] = 下周 lag_1 = 本周实际值
train = train.dropna(subset=["_y"])

target_cols = [c for c in train.columns if c.startswith("target_") or c == "_y"]
X_train = train.drop(columns=["data_date"] + target_cols, errors="ignore")
y_train = train["_y"]

目标函数 objective(trial)

Optuna 每次调用此函数即为一次 Trial。TPESampler 根据 trial.suggest_*() 定义的搜索空间推荐一组超参，在训练集前 90% 上训练、后 10% 上验证，返回 1 − MDA 供 Optuna 最小化。

🎯 objective() — 核心优化目标 ▼ 
def objective(trial, X_train, y_train):
  # trial.suggest_* → CSV params_* 列（9 列）
  params = {
    "num_leaves":       trial.suggest_int("num_leaves", 15, 255),
    "max_depth":        trial.suggest_int("max_depth", 3, 15),
    "learning_rate":    trial.suggest_float("learning_rate", 0.01, 0.3, log=True),
    "n_estimators":     trial.suggest_int("n_estimators", 100, 2000),
    "min_child_samples": trial.suggest_int("min_child_samples", 5, 100),
    "subsample":        trial.suggest_float("subsample", 0.5, 1.0),
    "colsample_bytree": trial.suggest_float("colsample_bytree", 0.5, 1.0),
    "reg_alpha":        trial.suggest_float("reg_alpha", 1e-8, 10.0, log=True),
    "reg_lambda":       trial.suggest_float("reg_lambda", 1e-8, 10.0, log=True),
    "random_state": SEED, "deterministic": True, "verbosity": -1, "n_jobs": -1,
  }
  split_idx = int(len(X_train) * 0.9)    # 前 90% 训练，后 10% 验证
  model = lgb.LGBMRegressor(**params)
  model.fit(X_train.iloc[:split_idx], y_train.iloc[:split_idx])
  y_pred = model.predict(X_train.iloc[split_idx:])

  mda = mean_directional_accuracy(y_val, y_pred)
  trial.set_user_attr("mda", float(mda))   # → user_attrs_mda
  trial.set_user_attr("mape", float(mape)) # → user_attrs_mape
  return 1.0 - mda                         # → value 列

搜索的 9 个超参数

覆盖 LGBM 的树结构、训练策略、正则化三大维度。其中 learning_rate 和 reg_* 使用 log-uniform 分布——在低值区更密集采样，因为小数变化对效果影响更大。

参数范围分布作用
num_leaves15 ~ 255整数均匀叶节点数，控制树复杂度
max_depth3 ~ 15整数均匀树深度上限
learning_rate0.01 ~ 0.3log-uniform学习率
n_estimators100 ~ 2000整数均匀树的数量
min_child_samples5 ~ 100整数均匀叶节点最少样本数（正则）
subsample0.5 ~ 1.0浮点均匀数据行采样比例
colsample_bytree0.5 ~ 1.0浮点均匀特征列采样比例
reg_alpha1e-8 ~ 10.0log-uniformL1 正则化
reg_lambda1e-8 ~ 10.0log-uniformL2 正则化

六、关键设计决策

决策项选择原因
优化目标MDA（方向准确率）库存预测核心诉求是涨跌方向
试验次数200 次先 50 验证再 200 正式；实测 100 次后边际收益归零
采样器TPESampler（贝叶斯）利用历史结果指导搜索，比随机收敛更快
剪枝器无（周频版移除）验证集仅 ~40 行，剪枝不可靠
验证方式train 后 10% 临时拆分不设独立 val.csv。每轮 trial 从 train 内部按时间顺序取最后 10% 做验证，仅用于 Optuna 打分
最终评估独立 test_weekly.csv调参全程不加载；只在复现推理阶段用最优参数在 train 全量重训后首次评估
数据隔离14 周 Gap防时间泄露
可复现性SEED=42 全局统一训练 + 拆分 + Optuna 全锁定

8.3 评估对比

三个阶段评估的区别：
「训练集评估」：在用于训练的全量 train_weekly.csv 上评估 —— 显示模型对见过的数据拟合程度。
「调参验证」：Optuna 每轮 trial 在 train 内部 90%/10% 临时拆分的评估——调参阶段使用，用来指导搜索方向。
「测试评估」：复现推理阶段，用最优参数在 train 全量上重训后，首次在独立 test_weekly.csv 上评估——这才是模型真正的泛化能力。
全程只有两份磁盘文件：train 和 test，没有独立的 val.csv。
