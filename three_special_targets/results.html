def winsorize_series(series, method="std", n_std=3):
    mean_val = series.mean()
    std_val = series.std()
    lower = mean_val - n_std * std_val
    upper = mean_val + n_std * std_val
    return series.clip(lower, upper)

def mean_directional_accuracy(y_true, y_pred):
    actual_dir = np.sign(y_true[1:] - y_true[:-1])
    pred_dir = np.sign(y_pred[1:] - y_pred[:-1])
    return np.mean(actual_dir == pred_dir)

MAPE数值差异说明

🔍 核心发现:

  • 两个流程的MAPE数值其实完全相同！ (6.42, 6.53, 5.12)
  • 但这个数值代表的是 642%, 653%, 512%，而不是 6.42%!
  • sklearn的 mean_absolute_percentage_error 返回值是小数形式，不乘以100

MAPE数值详解

标的ID 记录数值 实际百分比 说明
ID01517011 6.421969 642.20% 预测误差很大，平均偏差6倍多
ID01516873 6.528999 652.90% 预测误差很大，平均偏差6.5倍
ID01517220 5.118302 511.83% 预测误差很大，平均偏差5倍多

为什么MAPE这么高？

原因分析 (以ID01517220为例):

  • 测试集真实值: 从63.26 → 20.99，呈现快速下降趋势
  • 模型预测值: 基本在28-30附近徘徊，没有跟随下降趋势
  • 第一个样本: 真实值63.26，预测值28.70，APE=54.6%
  • 后续样本: 真实值持续下降，但预测值调整滞后
  • 当真实值变小时: 较小的绝对误差也会导致很大的百分比误差
